File Permissions in Linux

In my previous roles, I worked extensively with Linux systems to ensure proper access control and security of files and directories. I used Linux commands to analyze, modify, and enforce file permissions to align with organizational security policies.

Examples of Linux Commands

  1. Check File and Directory Permissions

    To analyze permissions for files and directories in the projects folder, I used the ls -la command to view a detailed list of their permissions. This helped identify misconfigured access rights.

    • Command Example: ls -la
    • Outcome: Identified that project_k.txt had write access for "other," which was against organizational policies.
  2. Remove Write Permissions for "Other"

    Upon identifying that project_k.txt allowed write access for "other," I used the chmod command to remove these permissions and verified the changes with ls -la.

    • Command Example:
      • chmod o-w project_k.txt
      • ls -la
    • Outcome: Ensured that only the user and group could modify the file, restricting "other" to read-only access.
  3. Modify Permissions on a Hidden File

    For the archived file .project_x.txt, I adjusted permissions so that no one had write access, while the user and group retained read access.

    • Command Example:
      • chmod u-w .project_x.txt
      • chmod g-w .project_x.txt
      • chmod g+r .project_x.txt
    • Outcome: Secured the hidden file by removing all write access, ensuring data integrity.
  4. Restrict Access to a Directory

    To secure the drafts directory and its contents, I restricted execute permissions for "group" and "other," ensuring only the researcher2 user could access it.

    • Command Example: chmod go-x drafts
    • Outcome: Restricted access to the directory, ensuring only authorized users could interact with its contents.

Summary

Using Linux commands like ls -la and chmod, I enforced access control policies and ensured the security of sensitive data within the projects directory. These commands allowed me to effectively identify and rectify permission issues, ensuring compliance with organizational security standards.
