My Projects and Certifications

  • Risk Register

    A detailed document identifying potential risks to a bank located in a coastal area, along with their likelihood, severity, and priority. Examples include compromised user databases, financial records leaks, and supply chain disruptions. The risk register includes mitigation strategies to strengthen the bank's overall security posture.

  • Vulnerability Assessment Report

    An in-depth assessment of a database server running on Linux and MySQL, identifying potential threats such as sensitive data exfiltration and system disruption. The report includes a risk assessment matrix and a remediation strategy, emphasizing encryption, access controls, and monitoring tools.

  • Incident Handler's Journal

    Documentation of a cybersecurity incident involving ransomware, describing the attackers' methods and motivations. The journal entry includes an analysis of the event, recommendations for preventing future attacks (e.g., implementing MFA and employee training), and a discussion on incident response strategies.

  • Automating IP Address Management for Access Control

    A Python-based automation project to update an IP address allow list by removing unauthorized addresses. The algorithm utilizes file I/O operations, list manipulation, and conditionals to streamline the process of managing access controls in a secure and efficient manner.

  • Access Control Worksheet

    A practical exercise in identifying and addressing access control issues, based on a scenario where all employees had unrestricted access to files at a company. Recommendations include implementing role-based access control (RBAC), enforcing least privilege policies, and conducting regular audits.

  • Controls Assessment Checklist

    An assessment of organizational controls and compliance practices, focusing on areas like least privilege, disaster recovery, encryption, and GDPR adherence. The checklist evaluates gaps and provides actionable recommendations to enhance security and align with industry standards.

  • Apply Filters to SQL Queries

    A project demonstrating the use of SQL queries to filter and analyze data for security purposes. Examples include identifying after-hours login attempts, isolating specific user activities, and retrieving departmental employee data for updates. The project emphasizes efficient data handling and actionable insights.

  • File Permissions in Linux

    A Linux-based project focusing on securing files and directories by managing permissions. Examples include removing unnecessary write access, securing hidden files, and restricting directory access using commands like ls -la and chmod. The project highlights the implementation of least privilege and data protection.
